# 第三章 深度学习

## 章节概述

本章通过分析经典深度学习算法，学习将基础神经网络应用到实际场景的方法，并逐步优化实现工业级应用。主要内容涵盖：适合图像处理的卷积神经网络（CNN）、经典CNN架构（AlexNet、VGG、Inception、ResNet）、目标检测算法（R-CNN系列、YOLO、SSD）、图像生成（GAN）、适合序列处理的循环神经网络（RNN/LSTM/GRU）、注意力机制与Transformer架构、从深度学习到大模型的演进、以及神经网络量化技术。

## 3.1 适合图像处理的卷积神经网络（CNN）

### 3.1.1 为什么需要CNN

- **全连接网络的问题**：处理大图像时参数过多。例如1024x1024图像，第一层1000个隐层神经元，第一层权重数量级达10^9，会导致过拟合。
- **CNN的解决方案**：通过**局部连接**和**权重共享**两个核心特性有效减少权重数量。
  - 全连接参数量：w_i x h_i x w_o x h_o
  - 卷积参数量：f x f（仅为卷积核大小）

### 3.1.2 CNN的基本组成（以VGG16为例）

| 组件 | 作用 |
|------|------|
| **卷积层（Conv Layer）** | 提取特征，检测边缘/纹理/形状 |
| **池化层（Pooling Layer）** | 下采样，减小特征图尺寸，控制过拟合 |
| **全连接层（FC Layer）** | 分类器，将高维特征映射为各类别概率 |
| **Softmax** | 输出归一化分类概率 |

VGG16结构：image(224x224) -> conv1(64)x2 -> maxpool -> conv2(128)x2 -> maxpool -> conv3(256)x3 -> maxpool -> conv4(512)x3 -> maxpool -> conv5(512)x3 -> maxpool -> FC4096 -> FC4096 -> FC1000 -> softmax

### 3.1.3 卷积运算详解

**数学定义**：y(n) = sum_i(x(i) * h(n-i)) = x(n) * h(n)，其中"*"表示卷积。

**神经网络中的卷积**：实际为计算矩阵内积（相关系数）。将卷积核在输入特征图上滑动，计算每个位置的加权和。

**多输入通道单输出特征图的卷积**：
- 输入：6x6x3（3通道特征图）
- 卷积核：3x3x3（每个通道对应一个3x3核）
- 输出：4x4（各通道卷积结果求和）

**多输入通道多输出特征图的卷积**：
- 使用多个卷积核（如2个3x3x3卷积核）
- 输出：4x4x2
- **不同的滤波器可检测不同特征**（如垂直边缘、对角线边缘等）

**卷积运算可转换为矩阵相乘**：将卷积的相乘再相加过程转换为向量内积，多输入输出通道卷积可转换为矩阵相乘，提高计算效率。

### 3.1.4 卷积层参数详解

| 参数 | 含义 |
|------|------|
| 输入特征图 | W_i x H_i x C_i |
| 卷积核 | C_o x K x K x C_i（可训练参数） |
| 偏置（bias） | 1 x 1 x C_o（可训练，使分类器偏离激活函数原点更灵活） |
| 输出特征图 | W_o x H_o x C_o |
| stride | 卷积窗口滑动步长 |
| pad | 边界扩充大小 |

**输出尺寸公式**：`output = ((W_i + 2p - K) / s + 1) x ((H_i + 2p - K) / s + 1)`

**计算量**：W_o x H_o x C_o x C_i x K x K

**边界扩充（Padding）**的作用：
- 扩大输入图像/特征图尺寸并填充像素
- 防止深度神经网络中特征图被动持续减小
- 强化图像边缘信息

### 3.1.5 深度可分离卷积（Depthwise Separable Convolution）

- **参数量**：K x K x C_i + C_i x C_o
- **计算量**：W_i x H_i x K x K x C_i + W_o x H_o x C_i x C_o
- 相比标准卷积大幅减少参数和计算量，用于MobileNet等轻量网络

### 3.1.6 池化层（Pooling Layer）

- **类型**：Max Pooling / Average Pooling / L2 Pooling
- **作用**：主动减小图片尺寸，减少参数数量和计算量，控制过拟合
- **特点**：不引入额外参数
- **Max Pooling**：保留特征最大值，提高提取特征的鲁棒性

### 3.1.7 全连接层与Softmax

- **全连接层**：将特征提取得到的高维特征图映射为一维特征向量，转化为各个类别的概率
- **Softmax公式**：`f(z_j) = e^(z_j) / sum_i(e^(z_i))`
- 作用：对输出进行归一化，输出分类概率；凸显最大分量，抑制远低于最大值的其他分量
- Softmax层的输入、输出数据规模相同

### 3.1.8 CNN的层排列规律

常见排列方式：`[Conv + ReLU] x N -> [Pool] x M -> [FC + ReLU] x P -> FC/Softmax`

例如N=3, M=1, P=2：input -> ConvReLU -> ConvReLU -> ConvReLU -> Pool -> FC(ReLU) -> FC(ReLU) -> FC -> output

### 3.1.9 深度 vs 广度

- **为何选择"深"而非"广"**：
  - 深度网络可从局部到整体"理解图像"：浅层卷积层感受野小，学习局部特征；深层卷积层感受野大，学习整体特征
  - 深度网络可减少权重数量：多个小卷积替代一个大卷积，获得更多样特征的同时权重更少

### 3.1.10 CNN的反向传播训练

CNN各层的反向传播（以 N=1, M=1, P=1 为例）：

1. **Softmax层反向传播**：
   - 交叉熵损失：L = -sum_i(y_i * ln(ŷ_i))
   - 梯度：grad_C2(L) = dL/dC^(2) = ŷ - y
   - Mini-batch下：grad_C2(L) = (1/p) * (Ÿ - Y)

2. **ReLU层反向传播**：
   - 梯度：若S>=0则传递梯度，否则为0
   - grad_S(i)(L) = grad_C(i)(L)（当S(i)>=0），否则0

3. **全连接层反向传播**：
   - grad_W(2)(L) = f * (grad_S(2)(L))^T
   - grad_b(2)(L) = grad_S(2)(L)
   - grad_f(L) = W^(2) * grad_S(2)(L)

4. **池化层反向传播**：
   - 找到最大值所在位置，仅将梯度传递给该位置
   - grad_C1(n,c,hs+q0,ws+q1)(L) = grad_P(n,c,h,w)(L)

5. **卷积层反向传播**：
   - 通过翻转卷积核进行转置卷积

## 3.2 经典CNN架构

### 3.2.1 AlexNet

- 2012年ImageNet冠军，首次在CNN中成功应用ReLU、Dropout
- 使用GPU训练，开创深度学习在计算机视觉的新纪元
- Top5错误率：15.3%

### 3.2.2 ZF-Net

- 在AlexNet基础上可视化了CNN各层学习到的特征
- 使用更小的卷积核和步长

### 3.2.3 VGGNet

**核心思想**：使用规整的小卷积核(3x3)堆叠，增加网络深度。

**VGG成功的原因**：
1. 更深的卷积网络 + 更多非线性激活函数，提升分类准确率
2. 使用规则的多层小卷积替代大卷积，减少参数数量，提高训练收敛速度
3. 部分网络层参数的预初始化，提高训练收敛速度

**规整的卷积-池化结构**：
- 所有卷积层：kernel=3x3, stride=1, pad=SAME
- 所有池化层：kernel=2x2, stride=2, pad=0
- 卷积层负责数据体深度变换（控制特征图数量），池化层负责长宽变换（控制特征图大小）

**多层小卷积 vs 单层大卷积**：
- 一个5x5卷积和两个3x3卷积的感受野相同
- 但两层3x3卷积有更多的ReLU，决策函数区分能力更强
- 实验结果：5x5卷积网络比两个3x3卷积网络top-1准确率低7%
- 参数更少：5x5需25个参数 vs 两个3x3需18个参数

**训练策略**：由简单到复杂，训练浅层网络后，用其权重作为深层网络的初始值。VGG16在ImageNet上Top5错误率：7.5%。

### 3.2.4 Inception系列（GoogLeNet）

**核心思想**：叠加多种尺寸的卷积层和池化层，获得不同尺度的特征。

**1x1卷积的作用**：
- 跨通道聚合，起降维（或升维）作用
- 形成"瓶颈层"，有效减少参数量和计算量
- 相当于在输入输出之间做特征全连接，提取非线性特征

**效率示例**（使用1x1瓶颈层前后对比）：
- 无瓶颈层：乘加次数约4.8x10^8，参数约6.1x10^5
- 有瓶颈层（1x1降维）：乘加次数约6.7x10^7，参数约8.5x10^4
- 计算量减少约7倍，参数量减少约7倍

**Inception系列发展**：

| 网络 | 主要创新 | Top5错误率 |
|------|----------|-----------|
| GoogLeNet(Inception-v1) | 提出Inception结构 | 6.67% |
| BN-Inception | Batch Normalization，用两个3x3代替5x5 | 4.82% |
| Inception-v3 | 二维卷积拆为两个一维卷积(1x3,3x1)，FC层加BN | 3.5% |
| Inception-v4 | Inception模块化，结合ResNet跳转结构 | 3.08% |

**辅助分类网络**：训练时，让中间某层输出经softmax得到分类结果，按较小权重加到最终分类结果中（模型融合），防止梯度消失。推断时辅助分类网络被去掉。

### 3.2.5 ResNet（残差网络）

**核心问题**：卷积层堆积就能提升准确率吗？实验发现56层网络比20层网络错误率更高（CIFAR-10和ImageNet）。

**原因分析**：不是梯度消失（BatchNorm可缓解），也不是过拟合（训练集上误差同样更高），而是**神经网络退化**——收敛到极值点而非最值点。

**残差学习**：
- Plain network：直接用多项式拟合输出 H(x)
- Residual network：用多项式拟合残差 F(x) = H(x) - x，最终输出 H(x) = F(x) + x
- 优点：对数据波动更灵敏，更容易学习获得最优解

**残差块结构**：`input -> Conv -> ReLU -> Conv -> +input(shortcut) -> ReLU -> output`

**ResNet网络改造规则**：
- 基本全由卷积层构成：kernel=3x3, stride=1, pad=SAME
- 特征图尺寸减小由stride=2的卷积层完成
- 若特征图尺寸不变，特征数量也不变；若尺寸减半，特征数量翻倍
- 跳转连接：实线连接（尺寸/数量不变直接相加），虚线连接（尺寸减半/数量翻倍需1x1卷积映射或补0）

**ResNet论文**：Deep Residual Learning for Image Recognition, CVPR 2016 (Best Paper)，作者何凯明等。ResNet152在ImageNet上Top5错误率仅3.57%。

**后续发展**：DenseNet（每层与前面所有层连接）、ResNeXt（分组卷积残差）等。

## 3.3 目标检测算法

### 3.3.1 目标检测概述

目标检测不仅需要分类图像内容，还需定位目标位置（Bounding Box）。分为两大类：
- **Two-stage检测**：先生成候选区域，再分类和位置回归
- **One-stage检测**：直接从图像预测边界框和类别

### 3.3.2 R-CNN系列（Two-stage方法）

**R-CNN**：
1. Selective Search生成约2000个候选区域
2. 每个候选区域缩放到固定大小送入CNN提取特征
3. SVM分类 + 边界框回归
4. 缺点：每个候选区域都做一次CNN，计算冗余，速度极慢（单帧50s）

**Fast R-CNN** 改进：
1. 直接对整张图像做卷积，不再对每个候选区域分别做卷积（减少大量重复计算）
2. 用ROI Pooling对不同候选框的特征进行尺寸归一化
3. 将边界框回归器放进网络一起训练
4. 用Softmax代替SVM分类器
5. 缺点：候选区域提取仍使用Selective Search，耗时仍在该阶段（2~3s）

**Faster R-CNN** = RPN + Fast R-CNN：
1. 卷积层提取特征图，供RPN和Fast R-CNN共享（大大减小计算时间）
2. RPN层生成候选区域：用3x3卷积增强特征，然后分两路（softmax判断前/背景 + bbox回归确定位置），经NMS去除冗余
3. ROIPooling将不同尺寸候选框池化为相同维度
4. 分类与回归

**Anchor机制**：对feature map每个位置，考虑9个anchor（3种面积x3种长宽比），输出2k个得分（前/背景概率）+ 4k个位置参数。

| 网络 | mAP (VOC2012) | 单帧时间 |
|------|--------------|---------|
| R-CNN | 53.3% | 50s |
| Fast R-CNN | 65.7% | 2s |
| Faster R-CNN | 67.0% | 0.2s |

### 3.3.3 YOLO系列（One-stage方法）

**核心思想**：将目标检测转换为直接从图像中提取bounding boxes和类别概率的单一回归问题，只需"看一眼"(You Only Look Once)即可检测目标类别和位置。

**YOLO v1实现**：
- 将输入图像分为SxS个格子
- 每个格子预测B个Bounding Box，每个bbox包含5个值（x, y, w, h, confidence）
- confidence = Pr(Object) * IoU(pred|truth)
- 每个格子预测C个条件类别概率
- 最终输出tensor维度：S x S x (B*5 + C)，对于PASCAL VOC：7x7x30

**YOLO v1优点**：
1. 检测速度快（45 FPS），Fast YOLO达155 FPS
2. 背景误判少（使用全局信息做预测）
3. 泛化性好（可迁移到其他领域）

**YOLO v1缺点**：
1. 邻近物体检测精度低（每个cell只预测两个bbox和一个分类）
2. 损失函数设计过于简单
3. 直接预测bbox位置导致训练不易收敛

**YOLO后续版本**：
- YOLO v2(YOLO9000)：引入Anchor Box，改进为Darknet-19网络，使用卷积层替代全连接层
- YOLO v3：多尺度预测(FPN)，更好的基础分类网络(Darknet-53)，Sigmoid代替Softmax用于多标签分类
- YOLO v4~v9：持续优化检测精度和速度

### 3.3.4 SSD（Single Shot MultiBox Detector）

**核心思想**：基于YOLO的one-stage检测 + Faster R-CNN的anchor-box思想 + 特征金字塔多尺度预测。在保持检测速度的同时提高准确度。

**多尺度特征图检测**：
- 大特征图（浅层）：检测较小的目标（感受野小，可划分更多单元）
- 小特征图（深层）：检测较大的目标（感受野大）

**Anchor Box设计**：
- 每个点对应6个anchor boxes（2个正方形 + 4个不同宽高比的长方形）
- 对多层特征图上的默认框计算IoU，找到与真实框最匹配的框
- 第k层scale：介于S_min=0.2和S_max=0.9之间按层均匀分布

## 3.4 图像生成：生成对抗网络（GAN）

### 3.4.1 GAN概述

- 2014年，Ian Goodfellow提出
- Yann LeCun评价："20年来机器学习领域最酷的想法"
- 解决的问题：从训练样本中学习出新样本，为无监督学习提供算法框架

### 3.4.2 GAN模型结构

| 组件 | 角色 | 目标 |
|------|------|------|
| **生成器G（Generator）** | 伪装者 | 生成以假乱真的样本，使判别器输出接近0.5 |
| **判别器D（Discriminator）** | 警察 | 区分真样本（输出接近1）和生成样本（输出接近0） |

**训练过程**：交替迭代
1. 更新判别器权值，使其能分清真假样本
2. 更新生成器权值，使其生成的假样本被判别器识别为真样本

### 3.4.3 GAN的训练目标

GAN训练是**极小极大博弈问题（零和博弈）**：
- **优化判别器D**：真样本x输入时输出接近1；生成样本G(z)输入时输出接近0
- **优化生成器G**：生成样本被判别器判断为接近1，即(1-D(G(z)))越小越好

### 3.4.4 GAN的问题与应对

| 问题 | 原因 | 应对方法 |
|------|------|---------|
| **梯度消失** | 判别器高置信度判断生成样本为假时，生成器梯度消失 | 修改生成器代价函数 |
| **模式崩溃（Mode Collapse）** | 某类假样本成功骗过判别器后，生成器趋向生成相似样本 | WGAN：采用更平滑的损失函数 |

### 3.4.5 GAN的变体

- **DCGAN**：将GAN中全连接网络扩展到卷积神经网络
- **CGAN（条件GAN）**：加入条件控制生成
- **ResGAN**：结合ResNet用于图像恢复
- **SRGAN**：超分辨率重建
- **CycleGAN**：图像风格转换

## 3.5 循环神经网络（RNN）

### 3.5.1 RNN的基本原理

循环神经网络通过带自反馈的神经元处理任意长度的序列。

**核心公式**：`h(t) = f(W * h(t-1) + U * x(t) + b)`，其中f常用tanh或ReLU。

**三个关键特性**：
- **时序（Sequence）**：前后输入数据不独立，相互影响
- **循环（Recurrent）**：对每个输入的操作相同，每时刻共享参数W和U
- **记忆（Memory）**：隐藏层h(t)捕捉所有t时刻之前的信息

### 3.5.2 RNN的输入-输出结构

| 结构类型 | 应用场景 |
|----------|---------|
| 序列->向量 | 情感分析（Sentiment Analysis） |
| 向量->序列 | 图像描述（Image Captioning） |
| 序列->序列 | 机器翻译（Machine Translation） |
| 同步序列->序列 | 视频分类/标注（Video Classification） |

### 3.5.3 RNN的正向传播

- 初始时刻：h^(0)为初始零向量
- 一般时刻：h^(t) = f(W*h^(t-1) + U*x^(t) + b)
- 输出：o^(t) = V*h^(t) + c, ŷ^(t) = softmax(o^(t))
- W、U、V、b在时间序列上**共享参数**

### 3.5.4 BPTT（时间反向传播）

- 某时刻损失：L^(t) = -y_t * ln(ŷ^(t))
- 整个序列损失：L = sum(L^(t)) for t=1 to tau
- 梯度对W的偏导：dL/dW = sum_t(sum_k(dL^(t)/dŷ_t * dŷ_t/dh^(t) * dh^(t)/dh^(k) * dh^(k)/dW))
- 由于递归结构，长序列中的梯度计算非常复杂

### 3.5.5 RNN的梯度消失与梯度爆炸

- 由于递归结构，序列长时梯度呈指数级增长或衰减
- **梯度爆炸**：参数值过大时，长序列梯度->无穷
- **梯度消失**：参数值过小时，长序列梯度->0
- 实际中RNN只能学习短期依赖关系

**解决方案**：
- 梯度爆炸：**梯度截断（Gradient Clipping）**
- 梯度消失：**LSTM、GRU**等改进模型

**长期依赖问题示例**："I grew up in Italy... I speak fluent Italian."需要记住远距离的"Italy"上下文。

### 3.5.6 LSTM（长短期记忆网络）

**核心思想**：引入单元状态（Cell State）作为信息传送带贯穿整个链条，通过门控机制控制信息流。

**三个门控**：
- **遗忘门（Forget Gate）**：决定遗忘多少前一时刻单元状态
- **输入门（Input Gate）**：决定写入多少新信息到当前单元状态
- **输出门（Output Gate）**：决定输出多少当前单元状态

**LSTM变体**：
- **窥视孔连接（Peephole Connection）**：门值不仅取决于h(t-1)和x(t)，也取决于C(t-1)
- **耦合输入门和遗忘门**：不单独决定遗忘和新增，一起做决定

### 3.5.7 GRU（门控循环单元）

**与LSTM的差异**：
- 将单元状态和隐藏状态合并
- 将遗忘门和输入门合并为**更新门（Update Gate）**
- 仅保留**重置门（Reset Gate）**（决定保留多少历史信息）和更新门
- 无输出门

**对比**：GRU参数量更少，训练速度快；LSTM在数据足够时表征能力更强。

### 3.5.8 Seq2Seq（序列到序列模型）

- 由**编码器（Encoder）**和**解码器（Decoder）**组成
- 编码器根据输入数据生成语义编码（Context Vector）
- 解码器根据语义编码输出处理结果
- 适用于输入输出序列不等长的任务：机器翻译、对话系统、自动文摘

**传统Seq2Seq的问题**：
1. 固定长度的语义编码难以存储较长输入序列的所有信息
2. 语义编码中每个元素权重相同，无法区分重要程度

### 3.5.9 注意力机制（Attention Mechanism）

**核心思想**：抽取少量重要信息并聚焦于这些信息，忽略不重要信息。

**注意力机制的三个优势**：
1. 聚焦关键信息，使模型做出更准确判断
2. 参数少，计算和存储开销较小
3. 可**并行计算**，比RNN序列化计算速度快

**注意力计算过程**（以Seq2Seq为例）：
1. 计算相似度：`alpha_ij = softmax(a(s_{i-1}, h_j))`
2. 注意力汇聚：`c^(i) = sum_j(alpha_ij * h^(j))`

## 3.6 Transformer架构

### 3.6.1 缩放点积注意力（Scaled Dot-Product Attention）

**公式**：`Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V`

- **Q（Query）**：查询向量
- **K（Key）**：键向量
- **V（Value）**：值向量
- 除以sqrt(d_k)进行**尺度缩放**，防止点积过大导致softmax梯度消失

**计算过程**：
1. Q和K点乘，得到词与词之间的相关关系矩阵
2. 经过尺度变换、掩码（Mask）和Softmax归一化，得到权重矩阵
3. 权重矩阵与V相乘，得到注意力输出

### 3.6.2 自注意力（Self-Attention）

**定义**：Q、K、V都来源于同一组输入：`Q = I*W_Q, K = I*W_K, V = I*W_V`

**与传统注意力的区别**：
- 传统注意力：基于输入和输出计算权重，只能捕捉输入输出间关系
- 自注意力：基于输入自身计算权重，可捕捉输入**内部**的相关关系

### 3.6.3 自注意力 vs RNN的复杂度比较

| 指标 | 自注意力 | RNN |
|------|---------|-----|
| **时间复杂度** | O(n^2 * d) | O(n * d^2) |
| **序列操作数** | O(1)（可一次完成） | O(n)（需迭代n次） |
| **最大路径长度** | O(1)（任意两点直接相连） | O(n)（需迭代n次） |

自注意力的优势：路径短->相关性信息传播时丢失最少；可并行->效率高。RNN的问题：路径长->信息丢失严重，难以建立长距离依赖。

### 3.6.4 多头注意力（Multi-Head Attention）

**原理**：基于相同注意力机制学习不同知识，再进行组合。

**计算过程**：
1. 对QKV进行h组不同线性变换
2. 将h组变换后的QKV并行计算注意力
3. 将h组结果拼接后进行线性映射

`MultiHead(Q,K,V) = Concat(head_1, ..., head_h) * W^O`
其中 `head_i = Attention(Q*W_i^Q, K*W_i^K, V*W_i^V)`

### 3.6.5 Transformer整体结构

Transformer包含**编码器**和**解码器**两部分：

**编码器（Encoder）**：
- 由N=6个相同层组成
- 每层包含两个子层：多头自注意力 + 前馈网络
- 每个子层有残差连接和层归一化

**解码器（Decoder）**：
- 也由N=6个相同层组成
- 每层在编码器两个子层基础上增加一个注意力子层（对编码器输出的交叉注意力）
- 自注意力子层增加了**掩码（Mask）**操作，确保预测第i个位置时只能看到i之前的输出

**训练 vs 推理**：
- 训练时：解码器输入为编码器输出+已偏移一位的标记，可**并行**训练
- 推理时：没有标记输入，前一步输出作为当前步输入，需逐个位置解码，**无法并行**

### 3.6.6 位置编码（Positional Encoding）

Transformer使用正弦/余弦函数编码位置信息：
- `PE(pos, 2i) = sin(pos / 10000^(2i/d))`
- `PE(pos, 2i+1) = cos(pos / 10000^(2i/d))`

位置编码与词嵌入相加后输入网络，以引入序列中的位置信息。

### 3.6.7 Transformer配置参数

| 配置 | 层数N | 维度d | FFN维度d_ff | 注意力头h | d_k=d_v | 参数量 |
|------|-------|------|-----------|---------|--------|--------|
| Base | 6 | 512 | 2048 | 8 | 64 | 65M |
| Large | 6 | 1024 | 4096 | 16 | 64 | 213M |

## 3.7 从深度学习到大模型

### 3.7.1 GPT（Generative Pre-trained Transformer）

**结构特点**：
- 采用Transformer**解码器**部分
- 12层，12注意力头，768维，d_ff=3072
- 512 tokens上下文，64 batch size
- 进行**单向**的Block连接（只看上文）

**训练方式**：
- **预训练（Pre-training）**：在大规模无标记语料上进行无监督学习（给定前N-1个词，预测第N个词）
- **微调（Fine-tuning）**：对每个特定下游任务利用有标记数据进行微调

**应用场景**：文本自动完成、生成对话、文章摘要等

### 3.7.2 BERT（Bidirectional Encoder Representation from Transformers）

**结构特点**：
- 采用Transformer**编码器**部分
- 进行**双向**的Block连接（同时看上下文）
- 预训练方式：**完形填空（Masked LM）+ 判断两句话是否有上下文关系（NSP）**

### 3.7.3 GPT系列发展历程

| 模型 | 时间 | 特点 |
|------|------|------|
| GPT | 2018.6 | Transformer解码器，1.17亿参数，5GB语料，上下文512 |
| GPT-2 | 2019.2 | 扩大参数和训练数据规模 |
| GPT-3 | 2020.5 | 1750亿参数，涌现能力，少样本学习 |
| CodeX | 2021.8 | 代码生成能力 |
| InstructGPT | 2022.3 | 基于人类反馈的指令微调(RLHF) |
| ChatGPT | 2022.11 | 对话优化，引爆AI热潮 |
| GPT-4 | 2023.3 | 多模态能力，更强推理 |

## 3.8 神经网络量化

### 3.8.1 为什么需要量化

深度神经网络的规模不断增大，计算和存储成本越来越高：
- BLOOM-176B：1760亿参数，训练需384块A100 3.5个月
- GPT-4：约1.7万亿参数，2.5万块A100训练近100天，花费超1亿美元

### 3.8.2 什么是神经网络量化

神经网络量化：通过将高位宽数据（如FP32）离散化得到低位宽数据（如INT8）进行网络的表示和计算。

### 3.8.3 量化的优势

- 减少计算量
- 提升计算速度
- 减少计算能耗
- 减少存储空间
- 减小乘法器面积

### 3.8.4 量化方法分类

| 分类维度 | 类型 | 说明 |
|----------|------|------|
| **量化对称性** | 对称量化 | 零点是零点 |
| | 非对称量化 | 零点可以偏移 |
| **量化均匀性** | 均匀量化 | 量化间隔相同，计算效率高 |
| | 非均匀量化 | 拟合数据好但计算效率低 |
| **量化粒度** | 分层量化 | 每层共享一套量化参数 |
| | 分通道量化 | 每通道独立量化参数，拟合更好但计算更多 |
| **优化目标** | 训练后量化(PTQ) | 无需重新训练 |
| | 量化感知训练(QAT) | 训练时模拟量化效果 |

### 3.8.5 对称量化

- 量化：`Q = R / (2^p)`
- 反量化：`R̃ = Q * 2^p`
- 量化参数p：`p = log2(Z / (2^{n-1} - 1))`，其中Z = max(|R|)

### 3.8.6 非对称量化

- 量化：`Q = (R - B) / (2^p)`，其中B = (Z_min + Z_max)/2
- 反量化：`R̃ = Q * 2^p + B`

### 3.8.7 量化过程（前向传播INT8量化）

前向传播：将**激活**和**权重**都量化后（FP32->INT8）再计算卷积和全连接，计算结果反量化回FP32。

反向传播：将激活、权重、激活梯度都量化后（FP32->INT8），再计算卷积和全连接的反传。
