你现在正在构建 RAG 知识库。

你的任务：
将已经清洗完成的教材文本，
严格按照语义边界切分为知识条目，
并写入 JSON 文件。

源文本是项目根目录下的/data/clean_data/*.md，
目标是把切分结果添加到项目根目录下的/rag/embedding_knowledge.json中


当前源文本已经完成：

- 引用展开
- 指代消解
- 公式替换
- 图片描述补全
- 实验名称替换
- 图表引用展开
- Latex 修正
- 格式清洗

因此：

你不需要进行任何文本优化，
也不允许进行任何内容修改。

你的职责只有：

“保持原文不变，并进行合理切分”。

==================================================
【输出格式】
==================================================

输出 JSON 数组：

[
  {
    "npyID": "",
    "source": "",
    "content": ""
  }
]

字段要求：

- npyID
  唯一递增 ID。可以留空统一不填写，因为后续会由代码生成。

- source
  原始章节名称。
  如“第2章-神经网络设计实验.md”中的知识为“第2章-神经网络设计实验”


- content
  对应知识正文。

==================================================
【绝对禁止事项】
==================================================

禁止：

- 改写原文
- 总结原文
- 缩写原文
- 优化表达
- 调整语序
- 删除内容
- 增加内容
- 合并内容
- 修改术语
- 修改标点
- 修改空格
- 修改换行
- 修改 latex
- 修改代码
- 修改公式
- 修改标题
- 修改大小写
- 修改列表结构
- 修改编号
- 修复病句
- 自行纠错
- 添加解释
- 添加注释
- 添加上下文说明

禁止进行：

- 引用展开
- 指代替换
- 图片解释
- 公式补充
- 图表补充
- 算法补充

输入文本已经是最终正确版本。

你只能：
“按边界切分”。

==================================================
【切分目标】
==================================================

目标 chunk 大小：

- 中文约 600~1200 字
- 或约 512~1500 tokens

但：

“语义完整性”
优先级高于长度。

==================================================
【切分规则】
==================================================

必须遵守：

1.
保持原始文本顺序。

2.
保持原始段落结构。

3.
保持原始章节结构。

4.
不要拆开：

- 定义
- 定理
- 推导过程
- 实验步骤
- 算法流程
- 因果分析
- 列表
- 表格说明

5.
如果一个完整知识单元超过 1200 字，
允许超长但是不要过长。**绝对禁止**超过3000tokens或者4000字。

不要强行切分。

6.
不要把多个独立主题强行合并到一个 chunk。

7.
不排斥为了上下文连贯把某个文段同时切分到前后两个chunk中，但是重叠度请控制在200tokens以下。

==================================================
【Chunk 质量要求】
==================================================

每个 chunk 应：

- 语义完整
- 独立可检索
- 独立可理解
- 尽量减少跨 chunk 依赖

但：

禁止为了“独立可理解”
而修改原文。

==================================================
【标题规则】
==================================================

如果 chunk 从某小节开始：

保留对应标题。

例如：

## 3.2 卷积层

正文……

如果 chunk 从章节中间开始：

不要补充解释。

不要添加：

[章节路径]

不要新增任何说明文字。

保持原文即可。

==================================================
【JSON要求】
==================================================

必须输出：

合法 JSON。

禁止输出：

- markdown
- ```json
- 注释
- 解释
- 说明文字
- 多余文本

只能输出 JSON 本身。

==================================================
【最终原则】
==================================================

这是“文本切分任务”，
不是“文本编辑任务”。

核心原则：

“除切分外，不做任何修改。”
