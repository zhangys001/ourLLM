# 第4章 编程框架实践

在第2、3章中，我们使用高级编程语言Python实现了卷积、池化、ReLU等深度学习算法中的常用操作，并最终实现了非实时风格迁移算法。在深度学习算法中，诸如卷积、池化、全连接等基本操作会被大量、重复地使用，而编程框架将这些基本操作封装成了一系列组件，从而帮助程序员更简单地实现已有算法或设计新的算法。

目前常用的深度学习编程框架有十多种，而PyTorch是其中最主流、应用最广泛的编程框架之一。PyTorch向上提供了一系列高性能的应用程序编程接口（Application Programming Interface，API），能够高效地实现各类深度学习算法；向下能够运行在包括CPU、GPU和DLP等在内的多种硬件平台上，具有良好的跨平台特性。

本章首先以VGG19为例，介绍如何使用PyTorch在CPU及DLP平台上实现图像分类；之后介绍如何使用PyTorch在CPU及DLP平台上实现实时风格迁移算法的推断；随后介绍如何实现实时风格迁移算法的训练过程；最后介绍如何在PyTorch中新增用户自定义算子，并将其集成到已经训练好的风格迁移网络中。

## 4.1 基于VGG19实现图像分类

### 4.1.1 实验目的

掌握PyTorch编程框架的使用，能够在CPU平台上使用PyTorch编程框架实现基于VGG19网络的图像分类，并在深度学习处理器DLP平台上完成图像分类。具体包括：

1. 掌握使用PyTorch编程框架处理深度学习任务的流程；
2. 熟悉PyTorch中常用数据结构的使用方法；
3. 掌握PyTorch中常用API的使用方法，包括卷积、激活等相关操作；
4. 与第3章的实验比较，理解使用编程框架实现深度学习算法的便捷性及高效性。

实验工作量：约30行代码，约需2个小时。

### 4.1.2 背景介绍

#### 4.1.2.1 PyTorch简介

PyTorch是由Meta开发并于2017年11月开源的深度学习框架，适用于Python、C++等编程语言，可以用于部署并实施大规模机器学习模型。其在功能、性能、灵活性等方面具有诸多优势，支持大规模的神经网络模型，并能够支持深度学习算法在CPU、GPU和DLP等硬件平台上的部署。

PyTorch提供了一系列高性能的API，方便程序员高效地实现深度学习算法。以目前较为常用的卷积神经网络VGG19为例，对于每个卷积层，首先输入与权重做卷积运算，然后加上偏置，最后通过非线性激活函数ReLU输出。在第3.1节的实验中使用高级编程语言实现了上述操作，而在PyTorch中则提供了一系列封装好的API，可以方便地实现上述操作。

实现VGG网络所需的主要函数的使用方法及参数含义如下表所示。

**PyTorch中常用函数：**

| 函数名 | 功能描述 | 参数介绍 |
|--------|----------|----------|
| `torch.nn.Sequential(*args)` | 一个时序容器。Modules会以它们传入的顺序被添加到容器中，也可以传入一个OrderedDict。 | `*args`：不定参数 |
| `add_module(name, module)` | 将一个child module添加到当前module，被添加的module可以通过name属性来获取。 | `name`：子模块的名称；`module`：要添加到模块中的子模块 |
| `torch.nn.Conv2d(in_channels, out_channels, kernel_size, stride=1, padding=0, dilation=1, groups=1, bias=True)` | 用来实现二维卷积操作。 | `in_channels`：输入信号的通道；`out_channels`：卷积产生的通道；`kernel_size`：卷积核的尺寸；`stride`：卷积步长；`padding`：输入的每一条边补0的层数；`dilation`：扩张操作，控制卷积核元素之间的间距；`groups`：控制分组卷积，默认不分组；`bias`：是否添加偏置 |
| `torch.nn.ReLU(inplace=False)` | 对输入运用ReLU函数。 | `inplace`：选择是否进行覆盖运算 |
| `torch.nn.MaxPool2d(kernel_size, stride=None, padding=0, dilation=1, return_indices=False, ceil_mode=False)` | 对于输入信号的输入通道，提供2维最大池化操作。 | `kernel_size`：max pooling的窗口大小；`stride`：窗口移动的步长，默认值是kernel_size；`padding`：输入的每一条边补0的层数；`return_indices`：如果等于True，会返回输出最大值的序号；`ceil_mode`：如果等于True，计算输出信号大小时使用向上取整 |
| `torch.nn.Flatten(start_dim=1, end_dim=-1)` | 将连续的维度范围展平为张量。 | `start_dim`：第一维（default=1）；`end_dim`：最后一维（default=-1） |
| `torch.nn.Linear(in_features, out_features, bias=True)` | 对输入数据做线性变换：$Y = Ax + b$ | `in_features`：每个输入样本的大小；`out_features`：每个输出样本的大小；`bias`：若设置为False，这层不会学习偏置 |
| `torch.nn.Softmax` | 对n维输入张量运用Softmax函数，将张量的每个元素缩放到(0,1)区间且和为1。 | 输入：(N, L)；输出：(N, L) |

PyTorch使用Python作为开发语言，并支持如NumPy、SciPy、PIL、OpenCV等多个Python扩展程序库以高效处理多种类型数据的计算等工作。例如：当需要读取以.mat文件格式保存的网络参数时，通常会使用SciPy库中的`scipy.io`模块；而当需要做图像相关处理时，通常会使用图像库PIL库中的Image模块、或是使用OpenCV-Python来对数字图像进行处理操作。这两个模块中常用函数的使用方法及参数含义如下表所示。

**常用的Python扩展程序库的相关函数：**

| 函数名 | 功能描述 | 参数介绍 |
|--------|----------|----------|
| `scipy.io.loadmat(file_name, mdict=None, appendmat=True, **kwargs)` | 装载MATLAB文件(.mat)，返回以变量名为键、以加载的矩阵为值的字典，格式为(mat_dict: dict)。 | `file_name`：mat文件的名称；`mdict`：插入mat文件变量的字典；`appendmat`：为True表示将.mat扩展名添加到file_name之后 |
| `PIL.Image.open(fp, mode='r')` | 打开并识别给定的图像文件。 | `fp`：一个文件名（字符串）、pathlib.Path对象或一个文件对象，文件对象必须以二进制模式打开；`mode`：模式，必须是"r" |
| `cv2.imread(filename, flags=None)` | 从指定的文件读取图像。 | `filename`：需要打开图片的路径；`flags`：图像的通道和色彩信息（默认值为1） |

PyTorch使用`torch.save()`函数来保存一个序列化的目标到磁盘。该函数用了Python的pickle程序用于序列化。其中，模型、张量和文件夹都是可以用这个函数保存的目标类型。同时，PyTorch使用`torch.load()`函数来加载模型，该函数使用Python的解压工具（unpickling）来反序列化pickled object到对应存储设备上。`torch.nn.Module.load_state_dict`用于将参数字典加载到模型结构中。

PyTorch支持模型保存的格式包括.pt、.pth、.bin、.onnx，文件格式的区别如下表所示。

**PyTorch输出文件格式比较：**

| 格式 | 解释 | 适用场景 | 可对应的后缀 |
|------|------|----------|-------------|
| .pt或.pth | PyTorch的默认模型文件格式，用于保存和加载完整的PyTorch模型，包含模型的结构和参数等信息。 | 需要保存和加载完整的PyTorch模型的场景，例如在训练中保存最佳的模型或在部署中加载训练好的模型。 | .pt或.pth |
| .bin | 一种通用的二进制格式，可以用于保存和加载各种类型的模型和数据。 | 需要将PyTorch模型转换为通用的二进制格式的场景。 | .bin |
| ONNX | 一种通用的模型交换格式，可以用于将模型从一个深度学习框架转换到另一个深度学习框架或硬件平台。在PyTorch中，可以使用`torch.onnx.export`函数将PyTorch模型转换为ONNX格式。 | 需要将PyTorch模型转换为其他深度学习框架或硬件平台可用的格式的场景。 | .onnx |
| TorchScript | PyTorch提供的一种序列化和优化模型的方法，可以将PyTorch模型转换为一个序列化的程序，并使用JIT编译器对模型进行优化。在PyTorch中，可以使用`torch.jit.trace`或`torch.jit.script`函数将PyTorch模型转换为TorchScript格式。 | 需要将PyTorch模型序列化和优化，并在没有Python环境的情况下运行模型的场景。 | .pt或.pth |

#### 4.1.2.2 基于DLP的PyTorch

为了使PyTorch在DLP上更好地运行，对原生PyTorch做了进一步扩展。基于DLP的PyTorch对原生PyTorch的主要修改有：添加MLU设备、部分算子的分发方式、Torchvision访问权限和CATCH拓展包等。

**基于DLP的PyTorch训练模式：** PyTorch在DLP上进行训练时，使用CNNL作为后端，包括很多训练模式，如量化感知训练、分布式训练、混合精度训练、单机单卡浮点训练。其中，单机单卡浮点训练模式为PyTorch在DLP设备上所支持的基础训练模式，为该课程主要介绍的内容。量化感知训练支持QAT（Quantization Aware Training，量化感知训练），以提高推理时的量化精度。分布式训练支持对Tensor进行单机多卡和多机多卡的卡间集合通信，对网络进行单机多卡和多机多卡的分布式训练。混合精度训练能够实现部分算子以float类型计算，部分算子以half类型计算。

**基于DLP的PyTorch推理模式：** PyTorch在DLP上进行推理时，使用CNNL作为后端，可以使用float精度来进行推理。其优势是简单易用，无需量化，推理过程灵活可逐层调试。在推理过程中，常常需要将动态图转换为静态图，从而获得更好的模型运行性能。具体可以调用`torch.jit.trace`函数，传入模型和输入的tensor，生成一个`torch.jit.ScriptModule`的实例。

### 4.1.4 实验内容

利用PyTorch的API，实现第3.1节中基于VGG19进行图像分类的实验，运行平台包括CPU和DLP。其中，DLP平台采用CNNL后端进行推理的方式。最后比较两种平台实现的差异。

### 4.1.5 实验步骤

VGG19图像分类实验主要包含以下操作：生成.pth文件（generate_pth.py）、CPU上进行推理（evaluate_cpu.py）、DLP上进行推理（evaluate_cnnl_mlu.py）。

为了便于迭代开发，工程实现时采用模块化的方式来进行处理，本实验包含网络结构模块、.pth文件生成模块、数据加载模块、CPU推理模块、DLP推理模块。

#### 4.1.5.1 网络结构模块

为方便对比，采用与第3.1节相同的预训练模型及层命名，逐层定义需要执行的操作，每一层的输出作为下一层的输入，从而搭建起完整的VGG19网络。

VGG19网络结构实现中，使用`torch.nn.Sequential`容器按顺序组织各层：首先定义卷积层序列（包括Conv2d和ReLU激活），然后定义最大池化层（MaxPool2d），通过Flatten层将多维特征图展平为一维向量，最后经过三个全连接层（Linear）和Softmax层输出分类结果。各层的通道数、卷积核大小、步长等参数按照VGG19标准配置设置。

#### 4.1.5.2 .pth文件生成模块

模型中各参数的初始化值来自经过训练的.mat文件。生成.pth文件时，首先使用`scipy.io.loadmat`函数读取预训练的.mat权重文件，提取各层的权重和偏置参数，然后使用PyTorch的`torch.save`函数将模型结构和参数保存为.pth文件。

#### 4.1.5.3 数据加载模块

使用PIL库中的Image模块读取输入图像，对图像进行预处理（如缩放、归一化等操作），将其转换为符合PyTorch处理格式的Tensor。

#### 4.1.5.4 CPU推理模块

在CPU平台上，加载.pth文件中的模型参数，运行VGG19网络的前向传播过程，对比网络输出与第3.1节的输出结果。具体而言，使用`torch.load`加载.pth模型文件，调用`model.eval()`将模型设为评估模式，然后将预处理后的输入图像传入模型得到分类结果。

#### 4.1.5.5 DLP推理模块

在DLP平台上进行推理时，首先指定DLP设备，将模型加载到DLP设备上，然后运行推理过程得到分类结果。DLP推理使用CNNL作为后端，通过`torch.jit.trace`将模型转换为TorchScript格式以获得更好的运行性能。

#### 4.1.5.6 实验运行

在CPU平台上运行以下命令，将VGG19网络保存为.pth文件，并进行CPU上的推理：

```bash
# 生成.pth模型，并在CPU上进行推理
bash run_cpu.sh
```

在DLP平台上运行以下命令，在DLP上进行推理：

```bash
# 在DLP上使用Eager模式进行推理
bash run_mlu.sh
```

### 4.1.6 推荐实践内容

- **基础实践**：在CPU平台上正确实现输入图像的读入、卷积层和池化层的定义等。可以正确生成.pth文件。
- **进阶实践**：在CPU平台上正确加载.pth文件模型参数，实现CPU上的VGG19网络的推理；给定VGG19的网络参数值和输入图像，可以得到正确的Softmax层输出结果和正确的图像分类结果。
- **高级实践**：在DLP平台上正确实现VGG19网络的推理；给定VGG19的网络参数值和输入图像，可以得到正确的Softmax层输出结果和正确的图像分类结果；处理时间相比CPU平台具有明显的提升。
