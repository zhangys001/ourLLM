实验工作量：约71行代码，5小时。

## 8.3.2 背景介绍

### 8.3.2.1 Code Llama介绍

Code Llama是一系列基于Llama 2的大语言模型，用于代码生成、代码补全等，它在几个代码基准测试中达到开放模型的最先进性能。该项目目前发布了多个版本模型：

- **Code Llama**：一个用于代码生成任务的基础模型。
- **Code Llama-Python**：一个专门用于Python的版本。
- **Code Llama-Instruct**：一个用人类指令和自指导代码合成数据进行微调的版本。

模型特点：
- 分别具有7B、13B和34B的参数规格。
- 所有模型都是在16k token的序列上进行训练，并在最多包含100k token的输入上显示出改进。
- 7B和13B的Code Llama和Code Llama-Instruct变体支持基于周围内容的补全。

**Code Llama的训练流程主要包括四个步骤：**

1. **从基础模型进行代码训练（Code Training from Foundation Models）**：使用通用语言模型Llama 2作为基础模型进行训练。
2. **代码补全（Infilling）**：不但提供代码生成的能力，还有代码补全的能力，可以用来在IDE中进行代码补全或者进行文档的生成，且该能力所花费的训练代价比较低。
3. **长输入上下文（Long Input Context）**：通过修改RoPE的参数，提供了仓库级的代码生成能力，极大地扩展了原始Llama 2的context的长度，从4096提升到了100k。
4. **指令微调（Instruction Fine-tuning）**：指令微调可以显著提升有用性、安全性，从而更准确地回应指令性问题。

### 8.3.2.2 Hugging Face Transformers简介

Hugging Face Transformers是一个用于自然语言处理（NLP）的开源库，提供了各种预训练模型。这些模型被广泛应用于各种任务，如文本分类、命名实体识别、问答、文本生成等。Transformers库易于使用，可方便地集成到现有的深度学习框架，如PyTorch和TensorFlow。

### 8.3.5.1 基于transformers库的Code Llama推理模块

基于transformers库的Code Llama主要利用Hugging Face的transformers库，调用CodeLlama-13b-Instruct-hf模型进行代码生成，实现了从给定的代码提示生成代码的功能，并提供了生成时间、标记数量等信息。

**1) 模型加载和设置模块**

使用Hugging Face的transformers库加载预训练的Causal Language Model（CLM）：
- 通过`AutoTokenizer`加载指定ID的预训练分词器；
- 使用`AutoModelForCausalLM`加载相应的预训练CLM模型；
- 参数设置：使用MLU作为运行设备（`device_map="mlu"`），采用`torch.float16`数据类型，关闭安全张量使用；
- 将模型设置为评估模式（`model.eval()`）。

**2) 文本生成和性能统计模块**

- 定义一个包含函数描述的prompt，其中有一个占位符`<FILL_ME>`；
- 使用time库记录代码执行的起始时间点；
- 通过tokenizer将prompt转化为模型可接受的输入格式（input_ids），并在MLU上进行处理；
- 利用预训练模型`model.generate()`生成相应的文本输出（设置max_new_tokens参数），将输出从MLU转回CPU；
- 计算代码执行的时间差，输出文本的长度以及每秒生成的token数量。

**3) 结果处理和打印模块**

- 使用tokenizer将模型生成的文本output还原为自然语言；
- 将填充文本替换回原始prompt中的占位符`<FILL_ME>`；
- 展示填充完成后的完整文本。

### 8.3.5.2 Transformer模型构建模块

Transformer模型构建模块实现了一个高效的Transformer模型，具备位置编码、多头注意力机制、前馈神经网络等基本组件，可用于处理自然语言处理任务。主要包括以下子模块：

**1) 模型并行化与初始化模块**

实现了一个Transformer模型，并使用fairscale库进行模型并行化。通过数据类`ModelArgs`定义了模型的参数，包括维度（dim）、层数（n_layers）、头数（n_heads）、词汇量（vocab_size）、多头数（n_kv_heads）等。支持MLU和MPS设备。

**2) RMS归一化模块**

定义了一个带可学习参数的RMS归一化（RMSNorm）模块。RMS的运算公式为：

$$RMSNorm(x) = \frac{x}{\sqrt{\frac{1}{n}\sum_{i=1}^{n} x_i^2 + \epsilon}} \cdot \gamma$$

其中，$x$ 表示输入张量，$\epsilon$ 表示平滑项用于避免除以零，$\gamma$ 为可学习的缩放权重参数。该模块可以被集成到神经网络中，用于增强模型的数值稳定性和收敛速度。

**3) 函数工具模块**

包含以下关键函数：

- **precompute_freqs_cis函数**：用于预先计算旋转注意力机制中的频率。首先计算频率向量，然后根据给定的维度和结束位置生成频率矩阵，使用极坐标形式计算频率的复数表示。
- **reshape_for_broadcast函数**：用于调整频率矩阵的形状，以便与输入张量进行广播。
- **apply_rotary_emb函数**：实现旋转位置编码操作，将输入的查询和键张量与频率矩阵进行旋转。将输入张量转换为复数形式，执行旋转操作后再转换为实数形式。
- **repeat_kv函数**：用于重复键值张量以适应分组查询注意力机制中的多头操作。如果重复次数为1则直接返回，否则扩展维度并重复。

**4) Transformer组件模块**

包含Attention模块和FeedForward模块：
- **Attention模块**：实现了分组查询注意力机制（GQA），使用`repeat_kv`扩展键和值以匹配查询头数。通过`apply_rotary_emb`应用旋转位置编码。
- **FeedForward模块**：使用SwiGLU激活函数的三层前馈网络结构。

**5) Transformer整体构造模块**

TransformerBlock模块将注意力层和前馈网络层通过残差连接和RMSNorm组合，多个TransformerBlock串联构成完整的Transformer模型。模型包含token嵌入层、多个TransformerBlock层、最终的RMSNorm和线性输出层。

### 8.3.5.3 Llama代码生成模块

Llama代码生成模块（generation.py）实现了基于Llama模型的文本生成功能，核心是`generate`函数：

- 接收prompt_tokens作为输入，使用模型进行自回归生成；
- 支持设置最大生成长度（max_gen_len）、温度（temperature）、top_p采样等参数；
- 在每个生成步骤中，模型预测下一个token的logits，通过softmax和采样策略选择下一个token；
- 支持提前停止（遇到EOS token时停止该序列的生成）；
- 返回生成的token序列。

### 8.3.5.4 Code Llama代码生成模块

Code Llama代码生成模块（example_completion_mlu.py）实现了从自然语言提示生成代码的完整流程：

- 加载Code Llama模型和分词器；
- 定义代码生成提示（如"Write a function to compute factorial"）；
- 将提示转换为模型输入格式；
- 调用模型的generate方法生成代码；
- 解码输出token序列为可读的代码文本；
- 打印生成的代码和性能统计信息。

### 8.3.5.5 Code Llama代码补全模块

Code Llama代码补全模块（example_infilling_mlu.py）实现了代码补全功能：

- 使用特殊的填充格式标记需要补全的位置（`<FILL_ME>`占位符）；
- 将代码上下文（前缀和后缀）按照Code Llama的填充格式组织为输入；
- 模型在指定位置生成补全代码；
- 将生成的代码替换回原始占位符位置，展示完整的补全结果。

### 8.3.5.6 Code Llama指令微调模块

Code Llama指令微调模块（example_instructions_mlu.py）实现了按照自然语言指令生成代码：

- 使用Code Llama-Instruct模型，该模型经过指令微调；
- 按照模型要求的指令格式组织输入（使用特殊的指令标记封装用户请求）；
- 根据自然语言指令（如"Write a Python function to sort a list using quicksort"）生成相应的代码实现；
- 输出生成的代码和性能统计。
