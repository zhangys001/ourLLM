# 4.2 实时风格迁移的推断

## 4.2.1 实验目的

掌握如何使用PyTorch实现实时风格迁移算法中的图像转换网络的推断模块，并进行图像的风格迁移处理。具体包括：

1. 掌握使用PyTorch定义完整网络结构的方法；
2. 掌握使用PyTorch加载模型结构及参数的方法；
3. 以实时风格迁移算法为例，掌握在CPU平台上使用PyTorch进行神经网络推断的方法；
4. 掌握在DLP平台上使用PyTorch对模型进行神经网络推断的方法。

实验工作量：约50行代码，约需3个小时。

## 4.2.2 背景介绍

### 4.2.2.1 实时风格迁移推断算法

第3.3节实现了一个非实时的图像风格迁移算法。该算法对于不同的输入图像，需要做多次迭代训练以输出风格化图像，因此耗时较长，实时性差。为了提高处理速度，Johnson等人提出了一种实时的风格迁移算法。该实时风格迁移算法中包含特征提取网络和图像转换网络。其中，特征提取网络使用在ImageNet数据集上预训练好的VGG16网络（为了实验的一致性，这里改为VGG19网络），用于提取风格图像、内容图像和生成图像的特征，计算内容损失和风格损失。

实时风格迁移训练时，首先输入图像到图像转换网络生成风格化图像，再利用特征提取网络计算内容损失和风格损失，然后迭代地更新图像转换网络参数以最小化损失。实时风格迁移推断时，利用训练好的图像转换网络对任意输入图像做网络推断来生成风格迁移后的图像，该过程基本达到实时。

**图像转换网络的结构：** 该网络主要包含几部分：下采样卷积层、残差层、上采样卷积层以及输出层，通过这种方式有利于减小计算量。具体而言：

- **下采样卷积层**：由三个卷积层构成。第一层卷积输出通道为32，使用9×9卷积核，stride为1，padding为4；第二层卷积输出通道为64，使用3×3卷积核，stride为2，padding为1；第三层卷积输出通道为128，使用3×3卷积核，stride为2，padding为1。
- **残差层**：由五个残差块构成，每个残差块由2个kernel_size为3×3、stride为1、padding为1的卷积组成。
- **上采样层**：不再使用传统的转置卷积，而是先执行上采样的操作（scale_factor为2，mode为nearest），再执行卷积操作（3×3卷积核），这种方式可以避免"棋盘伪影"现象的发生。以上操作重复执行两次，第一次上采样后将通道从128降至64，第二次从64降至32。
- **输出层**：由一个卷积层（输出通道为3，使用9×9卷积核，padding为4）以及sigmoid激活函数构成。

除了输出层，所有非残差卷积层后面都加了实例归一化（Instance Normalization）和ReLU操作，输出层使用的是sigmoid激活函数；第一层和最后一层卷积使用9×9卷积核，其他卷积层都使用3×3卷积核；每个残差块中包含两层卷积。

**图像转换网络结构参数总览：**

| 网络层 | 实现方式 | 输出特征图大小 |
|--------|----------|---------------|
| 输入 | - | 3×512×512 |
| 下采样层1 | 卷积：out_channels=32, kernel_size=9×9, stride=1, padding=4, bias=False | 32×512×512 |
| 下采样层2 | 卷积：out_channels=64, kernel_size=3×3, stride=2, padding=1, bias=False | 64×256×256 |
| 下采样层3 | 卷积：out_channels=128, kernel_size=3×3, stride=2, padding=1, bias=False | 128×128×128 |
| 残差层（×5） | 残差块，由2个kernel_size=3×3, stride=1, padding=1, bias=False的卷积组成 | 128×128×128 |
| 上采样层1 | 上采样scale_factor=2, mode=nearest + 卷积out_channels=64, kernel_size=3×3, stride=1, padding=1 | 64×256×256 |
| 上采样层2 | 上采样scale_factor=2, mode=nearest + 卷积out_channels=32, kernel_size=3×3, stride=1, padding=1 | 32×512×512 |
| 输出层 | 卷积：out_channels=3, kernel_size=9×9, stride=1, padding=4 | 3×512×512 |

**残差块结构：** 图像转换网络中包含五个残差块。残差块的基本结构为：输入 $x$ 经过一个卷积层（即权重层），再做ReLU，然后经过另一个卷积层得到 $F(x)$，再加上 $x$ 得到输出 $H(x) = F(x) + x$，然后做ReLU得到残差块的最终输出 $y$。当输入 $x$ 的维度与卷积输出 $F(x)$ 的维度不同时，需要先对 $x$ 做恒等映射使二者维度一致，然后再加和。与常规残差网络不同的是，图像转化网络中的残差块每个卷积层后面均连接一个实例归一化。

与卷积神经网络相比，残差块增加了从输入到输出的直连（shortcut connection），其卷积拟合的是输出与输入的差（即残差）。由于输入和输出都做了批归一化，符合正态分布，因此输入和输出可以做减法。残差网络的优点是对数据波动更灵敏，更容易求得最优解，因此能够改善深层网络的训练。

**实例归一化：** 深度学习中常见的归一化策略包括批归一化、实例归一化、层归一化以及组归一化等。

批归一化层由谷歌的科学家在2015年提出，它使用多个样本做归一化，将输入归一化到加了参数的标准正态分布上。这样可以有效避免梯度爆炸或消失，从而训练出较深的神经网络。批归一化的计算公式为：

$$\mu_i = \frac{1}{HWT}\sum_{t=1}^{T}\sum_{k=1}^{H}\sum_{j=1}^{W} x_{tijk}$$

$$\sigma_i^2 = \frac{1}{HWT}\sum_{t=1}^{T}\sum_{k=1}^{H}\sum_{j=1}^{W} (x_{tijk} - \mu_i)^2$$

$$\hat{x}_{tijk} = \frac{x_{tijk} - \mu_i}{\sqrt{\sigma_i^2 + \epsilon}}$$

其中，$x_{tijk}$ 表示输入图像批（batch）中的第 $tijk$ 个元素，$k$、$j$ 分别表示其在H、W方向的序号，$t$ 表示输入图像在batch中的序号，$i$ 表示特征通道号，$\epsilon$ 是为了避免除数为0时所使用的微小正数。

图像转换网络中，每个卷积计算之后激活函数之前都插入了实例归一化层。这样做的主要原因是批归一化方法是对整个图像batch做归一化以保证数据分布的一致性，而在风格迁移算法中，由于迁移后的结果主要依赖于某个图像实例，所以对整个输入batch做归一化的方法并不适合。2017年Ulyanov等人针对实时风格迁移算法提出了实例归一化方法。不同于批归一化，实例归一化对每个样本的每个通道独立做归一化：

$$\mu_{ti} = \frac{1}{HW}\sum_{k=1}^{H}\sum_{j=1}^{W} x_{tijk}$$

$$\sigma_{ti}^2 = \frac{1}{HW}\sum_{k=1}^{H}\sum_{j=1}^{W} (x_{tijk} - \mu_{ti})^2$$

$$\hat{x}_{tijk} = \frac{x_{tijk} - \mu_{ti}}{\sqrt{\sigma_{ti}^2 + \epsilon}}$$

实例归一化方法对每个图像实例独立做归一化，从而保持每个图像实例之间的独立，在风格迁移算法上取得了较好的效果，比较显著地提升了生成图像的质量。因此本实验用实例归一化方法来替代批归一化方法。

### 4.2.2.2 基于DLP的PyTorch推理

基于DLP的PyTorch上推理模式的运行流程为：依次需要指定硬件设备，加载数据集，加载模型和权重，然后拷贝到device端进行算子的分发，并将模型转换为eval模式。接着，为网络模型输入创建CNNL tensor和对应的device端内存储空间，然后启动CNNL进行推理。

## 4.2.4 实验内容

实时风格迁移推断实验利用PyTorch加载预先保存好的实时风格迁移模型文件（.pth文件），在CPU和DLP平台上运行实时风格迁移的推断（即图像转换网络的推断）得到风格化图像，并比较两种平台在实现上和运行性能上的差异。其中，模型文件中已经保存了图像转换网络的参数。

## 4.2.5 实验步骤

实时风格迁移推断实验主要包括以下模块：数据加载模块、网络结构模块、CPU推理模块、DLP推理模块。

### 4.2.5.1 数据加载模块

数据加载模块读取zip文件中的数据，并将数据转换为图像格式，经过预处理，将图像转换为符合PyTorch处理格式的Tensor。

数据加载使用自定义的COCODataset数据集类，继承自`torch.utils.data.Dataset`。在`__init__`方法中读取zip文件中的图像数据；在`__getitem__`方法中对图像进行预处理变换（包括调整大小、转换为Tensor等），返回处理后的图像Tensor。

### 4.2.5.2 网络结构模块

网络结构模块定义图像转换网络，包括下采样卷积层、残差层、上采样卷积层和输出层。卷积层使用`torch.nn.Conv2d`，其后依次连接实例归一化（`torch.nn.InstanceNorm2d`）和ReLU激活。残差块中，输入经过"卷积→实例归一化→ReLU→卷积→实例归一化"，然后与shortcut连接相加后再经过ReLU。上采样使用`torch.nn.functional.interpolate`配合卷积操作实现。输出层使用卷积加Sigmoid激活。

### 4.2.5.3 CPU推理模块

CPU推理模块（evaluate_cpu.py）加载图像转换网络的.pth模型文件，将输入图像传入网络进行前向传播，得到风格化输出图像并保存。

### 4.2.5.4 DLP推理模块

DLP推理模块（evaluate_cnnl_mlu.py）在DLP平台上运行推理。首先指定MLU设备（`torch.device('mlu')`），将模型和数据拷贝到DLP设备上，然后运行前向传播得到风格化图像。

### 4.2.5.5 实验运行

完成代码实现后，在CPU上运行以下命令，实现实时风格迁移推断，将输入的内容图像转换为风格化图像，风格迁移后的图像保存在outCpu/目录下：

```bash
# 在CPU上进行推理
bash run_cpu.sh
```

在DLP上运行以下命令进行实时风格迁移模型推断，风格迁移图片将会保存在/out/mlu_cnnl_mfus/目录下：

```bash
# 在DLP上进行推理
bash run_mlu.sh
```
