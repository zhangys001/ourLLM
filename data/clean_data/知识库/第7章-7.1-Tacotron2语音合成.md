### 7.1.2 背景介绍

#### 7.1.2.1 语音合成综述

语音合成（一般指文本到语音合成Text-To-Speech Synthesis，TTS）旨在通过给定文本生成清晰自然的语音，一直以来都是人工智能领域的热门研究方向。经典的语音合成算法通常依次由以下三部分组成：

1. **文本分析**：对文本进行归一化、分词等处理；
2. **声学模型（Acoustic Model）**：将输入的文本信息转换为声学特征（通常是Mel频谱），既可以使用传统的概率模型（如HMM），也可使用基于CNN/RNN/Self-Attention等构建的网络模型；
3. **声码器（Vocoder）**：基于声学特征生成语音音频作为最终输出。这一部分既可以使用传统基于参数合成的声码器，也可使用WaveNet、WaveGlow、WaveGAN、WaveVAE等多种基于神经网络的声码器。

较早的研究者通常使用参数统计方法合成语音，此后声码器和声学模型逐步被深度模型取代（WaveNet、DeepVoice系列、Tacotron系列、FastSpeech系列、AdaSpeech系列等）。目前则出现了一些完全端到端的语音合成模型，典型的有Tacotron/Tacotron2模型、DeepVoice3模型、FastSpeech/FastSpeech2模型、VITS模型等。本实验将要关注的语音合成算法就是Tacotron2。

#### 7.1.2.2 Tacotron2算法

Tacotron2是一个可直接从文本合成语音的端到端语音合成模型，它使用了带有注意力机制的双层LSTM作为声学模型，并使用WaveNet作为其声码器（后改为使用WaveGlow），二者间使用的声学特征中介为Mel频谱。最终Tacotron2在MOS（Mean Opinion Score）指标上较以往模型取得了较好表现，合成的语音可与真人语音媲美。

Tacotron2的整体结构具体包括Encoder、Decoder、Vocoder三部分，下面逐模块作详细介绍。

**1) Encoder（编码器）**

Encoder部分的功能为将输入的英文串转为相应的特征表示。首先将原始输入归一化整理为只含字母+部分标点的形式（数字、缩写等需要特殊处理），随后逐字符指定512维可训练的Embedding向量。然后使用3层5×1核的Conv1D综合相近字符的信息，通过padding维持长度不变。每层的核数都是512，因此这部分输出大小和输入相同。Encoder的最后一层使用了一个单层双向的LSTM。双向LSTM就是同时使用两套隐状态传递方向相反的LSTM，相同时刻提供相同的输入，相同时刻的输出拼接即得双向LSTM的整体输出。由于某个字符的发音可以和其前后的字符都有关，因此使用双向LSTM是合理的选择。

**2) Decoder（解码器）**

Decoder部分能够将Encoder的输出转为相应的Mel谱。它在核心的双层LSTM中提出了位置敏感注意力（Location-Sensitive Attention），这一注意力机制使得模型能够在不同时间步骤对输入的不同部分分配不同的重要性权重，以更好地对齐相关信息。在Encoder中还加入了PreNet和PostNet等辅助网络进行相关处理。

位置敏感注意力在双层LSTM两层之间生效。其计算权重的思路为：首先定义可任意取值的能量函数，随后用激活函数将其归一化为权值。能量函数的计算使用的是加性注意力：

$$e = w^T \tanh(W s + V h + U(F * \alpha))$$

其中，$e$ 是待求的能量值，$s$ 是第一层LSTM的当前隐状态，$h$ 为Encoder对文本的编码输出，$\alpha$ 是用上一时刻注意力权值和历史注意力权值总和拼接得到的向量。而 $w$、$W$、$V$、$U$ 均为可学习的全连接层，$F$ 为可学习的卷积层。

原则上，只需要带有字符信息的 $h$ 和有上一时刻Mel谱信息的 $s$（第一层LSTM输入的一部分为Mel谱输入PreNet的结果）即可得到相对合理的（Content-based的）注意力权值。把历史权重信息 $\alpha$ 加入运算（使用混合注意力机制），主要目的在于相互校准，以便更好地对齐输入文本与声音输出。

**3) Vocoder（声码器）**

Vocoder的作用是将音频特征转换为语音输出，它使用了一个生成式模型WaveGlow。其思路为：考虑服从标准正态分布的随机向量 $z \sim \mathcal{N}(0, I)$，可以构造含参 $\theta$ 的变换 $f_0, f_1, \ldots, f_k$，使我们希望生成的特定随机变量 $x$ 满足 $x = f_0 \circ f_1 \circ \ldots \circ f_k(z)$，则可直接通过采样获取生成结果。且只要各个变换可逆，则训练时可用极大似然法对 $p_\theta(x)$ 进行优化：

$$\log p_\theta(x) = \log p(z) + \sum_{i=0}^{k} \log \left| \det \frac{\partial f_i}{\partial y_{i-1}} \right|$$

WaveGlow中，将各层的输入 $x$ 等分为两段 $x_a$、$x_b$，只有 $x_a$ 会输入不可逆环节WN，且 $x_a$ 会直接复制一份到输出，让 $x_b$ 来做实际的线性变换。这样反向求解时 $x_a$ 不变，$x_b$ 只经过了可逆的线性变换，即可满足要求。WaveGlow中的WN就是WaveNet，不同的是其dilation不会重置为1，且由于它并非在生成 $x_a$ 自身，因此不是因果的。

Mel谱按照和WaveNet中相似的方式作为条件插入WN的Gated Activation Units中。Affine Coupling Layer中同属前半或后半的特征之间不会有直接的影响，因此增设一个可逆的1×1卷积环节（即乘上一个可学习的矩阵 $W$。为令其可逆，可以初始化为正交矩阵）。

在训练时，已知正确的声音波形 $x$，则可从 $x$ 计算出 $z$，然后使用极大似然法对各参数进行优化。在生成时，可以直接在标准正态分布中采样得到 $z$，然后反向计算出 $x$ 即得生成结果。

### 7.1.5 实验步骤

#### 7.1.5.1 位置敏感注意力模块

位置敏感注意力机制是Tacotron2模型的重要组成部分，这里主要对model.py中的Attention模块及相关模块进行代码补充。这部分的实现主要包括两个模块：LocationLayer模块和Attention模块。

**1) LocationLayer模块**

该模块用于处理历史注意力权值，其输出对应能量函数中 $U(F * \alpha)^T$ 项。其中，ConvNorm、LinearNorm为加入权值初始化的普通卷积和全连接层，在前向推理时和正常卷积、全连接层没有区别。该模块使用一维卷积对历史累积注意力权重进行卷积操作，提取位置特征。

**2) Attention模块**

利用LocationLayer模块和query_layer、memory_layer、V（三者均为全连接层，分别代表公式中的 $w$、$V$、$W$），实现能量值的计算。query_layer处理当前LSTM隐状态 $s$，memory_layer处理Encoder输出 $h$，LocationLayer处理历史注意力权重 $\alpha$，三者相加后经过tanh激活函数，再通过线性层V映射为标量能量值。经过mask处理后，该值经过Softmax即得注意力权重。

Attention模块的核心实现如下：

```python
# model.py — Attention模块
class Attention(nn.Module):
    def __init__(self, attention_rnn_dim, embedding_dim,
                 attention_dim, attention_location_n_filters,
                 attention_location_kernel_size):
        super(Attention, self).__init__()
        self.query_layer = LinearNorm(attention_rnn_dim, attention_dim,
                                       bias=False, w_init_gain='tanh')
        self.memory_layer = LinearNorm(embedding_dim, attention_dim,
                                        bias=False, w_init_gain='tanh')
        self.V = LinearNorm(attention_dim, 1, bias=False)
        self.location_layer = LocationLayer(attention_location_n_filters,
                                             attention_location_kernel_size,
                                             attention_dim)

    def forward(self, attention_hidden_cell, memory, processed_memory,
                attention_weights_cat, mask):
        # 计算注意力能量：e = w^T tanh(W·s + V·h + U·(F*α))
        processed_query = self.query_layer(attention_hidden_cell.unsqueeze(0))
        processed_attention_weights = self.location_layer(attention_weights_cat)
        energies = self.V(torch.tanh(
            processed_query + processed_attention_weights + processed_memory
        ))
        energies = energies.squeeze(-1)
        # mask处理
        if mask is not None:
            energies.data.masked_fill_(mask, -float("inf"))
        # 归一化为注意力权重
        attention_weights = F.softmax(energies, dim=1)
        # 计算注意力上下文
        attention_context = torch.bmm(attention_weights.unsqueeze(1), memory)
        attention_context = attention_context.squeeze(1)
        return attention_context, attention_weights
```

#### 7.1.5.2 Encoder模块

编码器（Encoder）用于将输入序列转化为编码器输出。其由三个一维卷积块和一个双向LSTM组成：

- **卷积块**：在初始化时，通过循环创建了encoder_n_convolutions个卷积层，每个卷积层由一个一维卷积（kernel_size=5, padding=2以维持长度不变）和批归一化组成。这些卷积层的目的是对输入序列进行特征提取和降维，每层输出512维特征。
- **双向LSTM**：将经过卷积层处理的特征序列输入双向LSTM中，以获得更丰富的上下文信息。双向LSTM的隐藏层维度为encoder_embedding_dim的一半（256维），双向拼接后得到512维输出。

在推理过程中，输入序列（字符嵌入向量）通过一系列卷积层进行逐层特征提取，每层后接批归一化和ReLU激活；然后特征序列被转置，传入双向LSTM得到最终的Encoder输出。

#### 7.1.5.3 Decoder模块

Tacotron2的解码器（Decoder）模块实现较为复杂，它的功能是根据编码器的输出和注意力机制，生成梅尔频谱输出。它由多个函数构成：

1. **`__init__`函数**：用于初始化Decoder类的各个属性和模块。它接收一系列超参数（如mel_channels、n_frames_per_step、attention_dim、prenet_dim、max_decoder_steps、gate_threshold等），并构建了多个神经网络层和模块，包括PreNet（两层全连接）、注意力LSTM层、解码器LSTM层、线性投影层和门控层等。

2. **`get_go_frame`函数**：用于生成一个全零的输入帧，作为解码的起始输入。该帧的形状与mel频谱的单帧形状一致。

3. **`initialize_decoder_states`函数**：用于初始化解码过程中的各个状态，包括注意力隐藏状态、注意力单元状态、解码器隐藏状态、解码器单元状态、注意力权重、累积注意力权重、注意力上下文和已处理的memory。它会创建一系列的零张量作为初始状态。

4. **`parse_decoder_inputs`函数**：用于处理解码器的输入，即mel频谱。它将输入转置并重塑，以适应模型的输入要求。

5. **`parse_decoder_outputs`函数**：用于整理解码器的输出，包括mel输出、门控输出和注意力权重。它对输出进行一系列的转置和重塑操作。

6. **`decode`函数**：执行解码的操作，包括注意力计算、解码器状态更新等。它根据当前解码输入、注意力状态和其他信息计算解码的输出。具体流程为：PreNet处理输入→注意力LSTM→注意力计算→解码器LSTM→线性投影（mel输出+门控输出）。

7. **`forward`函数**：是模型的正向传播过程，用于训练阶段。它接收编码器的输出、解码器的输入（Ground Truth mel频谱）以及编码器输出的长度，然后循环执行解码步骤，直到满足终止条件。

8. **`infer`函数**：用于模型的推理阶段，它在无Ground Truth的情况下生成梅尔频谱输出、门控输出和注意力对齐矩阵。推理流程为：
   - 获取初始的解码器输入帧（由于没有Ground Truth，初始输入帧为全零帧）；
   - 根据编码器输出的长度计算mask，以屏蔽填充部分；
   - 初始化解码器的状态和注意力相关变量；
   - 在每一步的解码循环中，解码器根据注意力机制和编码器的输出计算注意力权重、注意力上下文，并生成当前步的梅尔频谱输出和门控输出；
   - 根据门控输出的阈值将其转换为二值向量（门控输出通过sigmoid后与gate_threshold比较），并更新未完成的状态和梅尔频谱长度；
   - 如果启用了提前停止策略，并且所有样本的门控输出均为零，则停止解码过程；
   - 返回最终的梅尔频谱输出、门控输出和注意力对齐矩阵。

```python
# model.py — Decoder infer方法核心逻辑
@torch.jit.export
def infer(self, memory, memory_lengths):
    # 生成全零的go_frame作为解码起始输入
    decoder_input = self.get_go_frame(memory)
    
    # 初始化掩码
    mask = get_mask_from_lengths(memory_lengths)
    
    # 初始化解码过程中的各个状态
    (attention_hidden, attention_cell, decoder_hidden,
     decoder_cell, attention_weights, attention_weights_cum,
     attention_context, processed_memory) = self.initialize_decoder_states(memory)
    
    mel_lengths = torch.zeros([memory.size(0)], dtype=torch.int32, device=memory.device)
    not_finished = torch.ones([memory.size(0)], dtype=torch.int32, device=memory.device)
    
    mel_outputs, gate_outputs, alignments = (torch.zeros(1), torch.zeros(1), torch.zeros(1))
    first_iter = True
    
    while True:
        # 使用PreNet对decoder_input进行预处理
        decoder_input = self.prenet(decoder_input)
        
        # 调用decode函数进行单步解码
        (mel_output, gate_output, attention_hidden,
         attention_cell, decoder_hidden, decoder_cell,
         attention_weights, attention_weights_cum,
         attention_context) = self.decode(
            decoder_input, attention_hidden, attention_cell,
            decoder_hidden, decoder_cell, attention_weights,
            attention_weights_cum, attention_context,
            memory, processed_memory, mask)
        
        if first_iter:
            mel_outputs = mel_output.unsqueeze(0)
            gate_outputs = gate_output
            alignments = attention_weights
            first_iter = False
        else:
            mel_outputs = torch.cat((mel_outputs, mel_output.unsqueeze(0)), dim=0)
            gate_outputs = torch.cat((gate_outputs, gate_output), dim=0)
            alignments = torch.cat((alignments, attention_weights), dim=0)
        
        # 门控判断是否停止
        dec = torch.le(torch.sigmoid(gate_output), self.gate_threshold).to(torch.int32).squeeze(1)
        not_finished = not_finished * dec
        mel_lengths += not_finished
        
        if self.early_stopping and torch.sum(not_finished) == 0:
            break
        if len(mel_outputs) == self.max_decoder_steps:
            print("Warning! Reached max decoder steps")
            break
        
        decoder_input = mel_output
    
    # 整理解码器的输出
    mel_outputs, gate_outputs, alignments = self.parse_decoder_outputs(
        mel_outputs, gate_outputs, alignments)
    
    return mel_outputs, gate_outputs, alignments, mel_lengths
```

#### 7.1.5.4 Tacotron2模块

Tacotron2模块用于语音的合成任务。该模块主要由以下函数构成：

1. **`__init__`**：用于初始化模型的各个组件和参数，包括Embedding层、Encoder、Decoder和PostNet。

2. **`parse_batch`函数**：用于解析输入批次数据，提取文本序列、输入长度、mel频谱和目标长度。

3. **`parse_output`函数**：用于解析模型输出，整理mel频谱、门控输出和对齐矩阵。

4. **`forward`函数**：定义了前向传播方法，用于训练阶段的模型前向计算：
   - 将输入序列通过嵌入层进行嵌入，将其转置为适合模型输入的形状。嵌入层将离散的输入符号转换为连续的向量表示；
   - 将嵌入后的输入序列传递给编码器（Encoder）模块，编码器对输入序列进行处理并生成编码器输出；
   - 将编码器输出作为输入传递给解码器（Decoder）模块进行解码，生成梅尔频谱输出（mel_outputs）、门控输出（gate_outputs）、注意力对齐矩阵（alignments）以及梅尔频谱的长度（mel_lengths）；
   - 将梅尔频谱输出通过后处理网络（PostNet）进行后处理，将梅尔频谱输出与后处理输出相加，得到最终的梅尔频谱输出（mel_outputs_postnet）。

5. **`infer`函数**：定义了推理方法，用于推理阶段的模型前向计算。该函数接收输入文本和输入长度作为参数，用于生成mel频谱、gate频谱和注意力权重。其操作与训练过程中的前向传播类似，但该方法中不需要目标mel频谱（因为在推理时没有目标）。最终返回处理后的mel频谱、mel的长度以及注意力权重。

#### 7.1.5.5 推理模块

推理模块（test_infer.py）主要用于执行DLP上基于PyTorch的Tacotron2文本转语音推理并记录性能数据以进行性能分析和优化。它通过解析命令行参数来配置推理过程：

- 解析命令行参数（模型路径、设备参数、批大小、输入长度等）；
- 加载预训练的Tacotron2模型和WaveGlow模型；
- 创建去噪器（Denoiser），根据设备参数选择CPU或MLU；
- 准备输入文本，进行预处理得到输入序列；
- **使用`torch.no_grad()`禁用梯度计算**，进行推理过程；
- 在推理循环中分别测量预处理时间、Tacotron2延迟、WaveGlow延迟、类型转换时间和数据传输时间；
- 记录性能统计：包括每秒处理mel数、延迟、吞吐量等指标；
- 将生成的音频保存为.wav文件。

性能测量使用自定义的MeasureTime上下文管理器，记录各阶段的耗时，并汇总到measurements_all字典中进行统计分析。

### 7.1.7 实验思考

1. Tacotron2语音合成实验使用的都是32位浮点数据（fp32）进行计算，暂未使用混合精度模型，倘若使用混合精度模型推理，会对推理速度有什么影响？
