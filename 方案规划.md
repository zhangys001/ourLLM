# 智能计算系统大作业：向量检索基线方案规划

## 1. 核心架构与演进路线
- **当前阶段 (纯检索基线)**：使用 **Qwen Embedding** 对输入问题编码，通过余弦相似度匹配本地知识库。采用**双层检索策略**：先查 `embedding_qa` 撞原题（拿满时间分）；未命中原题则查 `embedding_knowledge` 提取最相关知识片段（大多数题目得分的“正主”）。
- **后续演进 (RAG 增强生成)**：引入 **Qwen2-1.5B**。用 `embedding_knowledge` 检出的多条关联知识片段组合 Prompt，交由大语言模型推理生成，以提升隐藏题的质量分。

## 2. 知识库数据结构
采用 JSON 存储文本与元数据，NPY 存储特征向量。NPY 文件通过 `tools/make_npy.py` 离线生成，不入 Git 库。

### 2.1 核心知识库 (`embedding_knowledge.json`)
由课程文档清洗切片而成，是解答未知问题的核心支撑。
```json
[
  {
    "npyID": "0", //由脚本自动填充
    "source": "第2章-神经网络设计实验", //源自文件名，可以作为tag写到prompt中
    "content": "具体的课程知识点内容块..."
  }
]
```

### 2.2 辅助 QA 库 (`embedding_qa.json`)
由公开思考题与标准答案构成，专用于精确匹配已知原题。
```json
[
  {
    "npyID": "0", //由脚本自动填充
    "Question": "明确的问题文本...",
    "Answer": "直接可用的标准答案..."
  }
]
```

## 3. 向量化与检索计算方案
- **文本编码 (Embedding)**：输入文本通过 tokenizer 后，取模型（Qwen Embedding）输出结果的 **last token**（最后一个词元）的隐藏层状态作为句向量表示。
- **降维与存储**：对提取的向量进行 **L2 归一化**（`torch.nn.functional.normalize(p=2)`），转换为 `float32` 格式的 NumPy 扁平数组保存至 `.npy` 文件中，形成候选矩阵。
- **相似度检索**：
  1. 实时将输入问题用同等逻辑转换为查询归一化向量 $q$。
  2. 利用矩阵乘法 $Candidates \times q$ 快速计算余弦相似度分数。
  3. **分层裁决**：若在 QA 库中相似度极高（例如 $>0.9$），直接输出对应的 `Answer`；否则在 Knowledge 库中取 Top 相似度的 `content`。未超阈值（如 $0.7$）则触发兜底回复。

## 4. 环境约束与打包规范
- **运行依赖**：完全符合评测机 Ubuntu 22.04, Python 3.12.3, PyTorch 2.8.0, Transformers 5.6.2，运行期间禁止外部 API 及联网。
- **ZIP 提交内容**：
  ```text
  model.zip
  ├── main.py                    # 单入口文件
  ├── model/
  │   ├── qwen_embedding_model/  # 编码模型权重
  │   └── qwen_model/            # Qwen2-1.5B 权重（后续RAG阶段用）
  └── rag/                       # 必须包含离线生成的 *.npy 和 *.json
  ```
- **注意**：`data/` (原始语料) 和 `tools/` (构建脚本) 严禁打包，确保压缩包精简且在 5GB 以内。
