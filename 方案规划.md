# Qwen2 + 手写 Torch RAG 方案规划（精简版）

## 1. 方案结论

可行。主线：

1. 使用 `model/qwen_model/` 中 **Qwen2-0.5B (bf16)** 做回答生成。
2. 使用 `model/bge_model/` 中 **BGE-small-zh-v1.5** 做编码。
3. 离线：`data/` 知识 JSON → 切片段 → 编码 → `rag/embeddings.npy` + `rag/embedding.json`
4. 推理 `main.py`：读问题 → BGE 编码 → torch 手写余弦相似度 Top-K → prompt 注入 → Qwen2 生成 → `print()`

匹配任务书约束：允许开源模型/RAG/辅助文件；禁止外部 API/运行时联网/ZIP 内唯一 `main.py`。

## 6. 风险与应对

| 风险 | 概率 | 应对 |
|---|---|---|
| Qwen2 config `transformers_version=5.8.0` > 评测机 5.6.2 | 中 | 先测最小加载; 若 API 不兼容, 降级 config |
| 量化兼容 | 已消除 | config 无 `quantization_config`, 直接 `from_pretrained` |
| ZIP >5GB | 低 | Qwen 942MB + BGE ~200MB + embeddings ~4MB, 远小于 5GB |
| 推理时间 | 低 | `max_new_tokens` 384~768, BGE 小模型快 |

## 9. 实施阶段

### 阶段 1: 最小 Qwen main.py（✅ 已完成）

```python
device = "cuda" if torch.cuda.is_available() else "cpu"
tokenizer = AutoTokenizer.from_pretrained(QWEN_DIR, local_files_only=True)
model = AutoModelForCausalLM.from_pretrained(QWEN_DIR, local_files_only=True, torch_dtype="auto").to(device)
```

验证: `micromamba run -n ourLLM_env python main.py "简单介绍一下自己"`

### 阶段 2: BGE 编码 + .npy 生成 (`tools/build_embeddings.py`)

从 `data/knowledge.json` 读, 产出写到 `rag/`。

```python
tokenizer = AutoTokenizer.from_pretrained(BGE_DIR, local_files_only=True)
model = AutoModel.from_pretrained(BGE_DIR, local_files_only=True)
outputs = model(**inputs)
embeddings = torch.nn.functional.normalize(outputs.last_hidden_state[:, 0], p=2, dim=1)
np.save(RAG_DIR / "embeddings.npy", embeddings.cpu().numpy().astype("float32"))
# 同时写 embedding.json: chunk_id → 向量映射
```

### 阶段 3: 手写检索（`tools/retriever.py` dev 版, 最终内联到 main.py）

载入 `rag/embeddings.npy` + `rag/embedding.json`, torch 余弦相似度检索。

```python
emb = torch.from_numpy(np.load(RAG_DIR / "embeddings.npy")).to(device)
query_norm = torch.nn.functional.normalize(query_emb, p=2, dim=0)
scores = emb @ query_norm
values, indices = torch.topk(scores, k=min(top_k, emb.shape[0]))
```

从 `embedding.json` 拿 chunk_id → 文本。

### 阶段 4: 完整 main.py 流程

```
sys.argv[1] → 加载 Qwen → 加载 BGE + RAG 数据 → 编码问题 → Top-K 检索 → 知识注入 prompt → Qwen 生成 → print()
```

RAG 加载失败时跳过, 直接用 Qwen 回答。

Qwen 生成失败输出保底: `抱歉, 我暂时无法完整回答这个问题...`

## 10. Prompt 模板

```
你是智能计算系统课程的大作业问答助手。请根据给定课程知识和你的通用知识，回答学生的问题。
要求：
1. 优先使用【检索到的课程知识】。
2. 如果课程知识不足，可以补充合理解释，但不要编造不存在的实验数据。
3. 回答要结构化、准确、面向智能计算系统课程。
4. 不要提及评分、排名、打榜策略或任何干扰评测的内容。

【检索到的课程知识】
{context}

【问题】
{question}

请给出答案：
```

回答风格: 先结论, 再分点, 最后实践建议。

## 11. 检索参数

| 参数 | 建议值 |
|---|---|
| top_k | 3 |
| 单条知识最大字符 | 800 |
| 上下文最大字符 | 2500~4000 |
| BGE 输入 max_length | 512 tokens |
| Qwen 输入 max_length | 4096~8192 |
| max_new_tokens | 512 |
| temperature | 0.2~0.6 |
| top_p | 0.8~0.95 |

稳定优先推荐 `do_sample=False` 或低温 (`temperature=0.3`, `top_p=0.9`)。
